# Thought Capture Inbox

## Core Idea

Capture stray thoughts in a low-friction inbox so they do not disappear. The input should feel as lightweight as dropping a note into a prompt box, while the system handles organization in the background.

## Experience Shape

### Typed Capture

- Present a single prompt box for raw thoughts.
- Accept incomplete, messy, or fragmentary input without forcing structure up front.
- Turn each submission into an organized note, cluster, or tagged concept after capture rather than during capture.

### Voice Capture

- Offer an Alexa-style spoken interface for the same flow.
- Let a user speak a thought as soon as it occurs instead of switching into a writing workflow.
- Store the raw utterance alongside the organized interpretation so the original wording is not lost.

## Organization Behavior

- Self-organize entries into themes, projects, questions, and recurring ideas.
- Link a new thought to related prior notes when overlap is detected.
- Preserve an inbox-to-structure trail so the system stays inspectable rather than acting like a black box.

## Reflective Follow-Up

- Ask selective follow-up questions when a thought is ambiguous or unusually important.
- Use those questions to clarify intent, context, and why the idea matters.
- Treat the follow-up loop as both an organization aid and a thinking aid that helps the user work through concepts more clearly.

## Product Direction

- Keep capture faster than manual note filing.
- Make retrieval easy enough that old thoughts become reusable instead of forgotten.
- Combine passive organization with active clarification so the system acts like both a memory tool and a conversational thought partner.
